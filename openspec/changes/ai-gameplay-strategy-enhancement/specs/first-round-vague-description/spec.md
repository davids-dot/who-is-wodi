## ADDED Requirements

### Requirement: 第一轮前三个玩家必须使用模糊描述
系统 SHALL 确保第一轮前三个描述的玩家使用模糊描述策略，不说具体功能，防止过早暴露词语。

#### Scenario: 第一轮第一个玩家描述
- **GIVEN** 游戏处于第1轮
- **AND** 当前玩家是第一个描述的人
- **WHEN** 构建描述 prompt
- **THEN** prompt 中 SHALL 包含"千万不要说得太具体"的约束
- **AND** 提示应该说"是一种日常用品"、"大家都见过"等模糊描述

#### Scenario: 第一轮第四个玩家可以正常描述
- **GIVEN** 游戏处于第1轮
- **AND** 当前玩家是第四个或之后描述的人
- **WHEN** 构建描述 prompt
- **THEN** 不添加模糊约束，允许正常描述
- **AND** 因为前面已有参考，可以基于他人描述进行描述

#### Scenario: 第二轮及以后正常描述
- **GIVEN** 游戏处于第2轮或之后
- **WHEN** 构建描述 prompt
- **THEN** 不添加第一轮的特殊约束
- **AND** 玩家可以正常描述

### Requirement: 模糊描述的具体要求
模糊描述 SHALL 遵循以下原则：
- 不说功能、用途
- 说属性、感觉、场景
- 例如："是一种日常用品"、"大家都见过"、"有黑白颜色的"

#### Scenario: 模糊描述示例
- **GIVEN** 玩家词是"键盘"
- **AND** 玩家是第一轮前三个描述的人
- **WHEN** 生成描述
- **THEN** 描述可以是"桌面常见物品"、"有黑白颜色的"
- **AND** 不能是"打字用的"、"敲代码必备"
